# IT Administrator: Software Rollout Checklist

**Role:** [IT Administrator / LegalTech Specialist](../../../roles/it-legaltech-specialist/README.md)  
**Frequency:** As needed (new tool deployment)  
**Approval:** COO or Managing Partner approves major purchases; IT leads rollout

---

## Scenario

Deploy a new firm-wide tool (e.g., Clio add-on, document management, e-signature) with minimal disruption and strong adoption.

---

## Prerequisites

- Tool selected and contract signed
- Budget approved
- Integration requirements identified (Clio, email, etc.)

---

## Step-by-Step

### 1. Pre-Rollout (2–4 weeks before)

| Step | Action |
|------|--------|
| 1.1 | Create project plan: timeline, milestones, owners |
| 1.2 | Set up staging/sandbox if available |
| 1.3 | Configure integrations (Clio API, SSO, etc.) |
| 1.4 | Prepare training materials (guides, videos, FAQs) |
| 1.5 | Identify champions (power users) per department |
| 1.6 | Communicate rollout date and expectations to staff |

### 2. Pilot (Optional, 1–2 weeks)

| Step | Action |
|------|--------|
| 2.1 | Deploy to pilot group (e.g., one practice area or team) |
| 2.2 | Gather feedback; fix issues |
| 2.3 | Refine training and configuration |
| 2.4 | Document lessons learned |

### 3. Full Deployment

| Step | Action |
|------|--------|
| 3.1 | Create user accounts; assign licenses |
| 3.2 | Configure role-based access if applicable |
| 3.3 | Migrate data if replacing existing tool |
| 3.4 | Verify integrations working |
| 3.5 | Go-live communication to all staff |

### 4. Training

| Step | Action |
|------|--------|
| 4.1 | Conduct live training sessions (or recorded) |
| 4.2 | Provide quick-reference guides |
| 4.3 | Offer office hours for questions (first 2 weeks) |
| 4.4 | Record attendance and completion |

### 5. Post-Rollout (Weeks 1–4)

| Step | Action |
|------|--------|
| 5.1 | Monitor support tickets; triage issues |
| 5.2 | Track adoption (logins, usage) |
| 5.3 | Address resistance; provide additional training |
| 5.4 | Report to COO: adoption rate, issues, next steps |

### 6. Decommission Old Tool (if applicable)

| Step | Action |
|------|--------|
| 6.1 | Confirm all data migrated |
| 6.2 | Archive old system per retention policy |
| 6.3 | Revoke access; cancel subscription |
| 6.4 | Document for future reference |

---

## Rollout Checklist (Printable)

**Pre-Rollout**
- [ ] Project plan created
- [ ] Staging configured
- [ ] Integrations tested
- [ ] Training materials ready
- [ ] Champions identified
- [ ] Staff notified

**Deployment**
- [ ] Accounts created
- [ ] Data migrated (if applicable)
- [ ] Integrations verified
- [ ] Go-live communication sent

**Training**
- [ ] Live sessions conducted
- [ ] Guides distributed
- [ ] Office hours scheduled

**Post-Rollout**
- [ ] Support triage in place
- [ ] Adoption tracked
- [ ] COO report prepared

## Tools

- **Clio** — Primary PMS; many add-ons integrate
- **Google Workspace / Microsoft 365** — SSO, email
- **Support ticketing** — Track issues during rollout
- **LMS or intranet** — Training materials, FAQs
